# Lab 07 – WMI Process Execution Detection (MITRE ATT&CK T1047)

## Objective

This lab demonstrates how attackers can abuse **Windows Management Instrumentation (WMI)** to execute processes. WMI is a legitimate Windows administration framework frequently abused by threat actors for **remote execution**, **lateral movement**, and **Living-off-the-Land (LotL)** techniques.

The objective of this lab is to simulate WMI-based process execution, observe the telemetry generated by Sysmon, and investigate the activity in Wazuh SIEM.

# Attack Scenario

An attacker abuses the native Windows **WMIC** utility to spawn a hidden PowerShell process.

This technique is commonly observed during:

- Lateral Movement
- Remote Administration
- Malware Execution
- Living-off-the-Land (LotL)

---

# Attack Commands

### Launch Notepad

```cmd
wmic process call create "notepad.exe"
```

### Launch Hidden PowerShell

```cmd
wmic process call create "powershell.exe -nop -w hidden"
```

# Sysmon Investigation

```powershell
Get-WinEvent -LogName "Microsoft-Windows-Sysmon/Operational" -MaxEvents 100 |
Where-Object {
    $_.Id -eq 1 -and
    ($_.Message -match "wmic.exe" -or $_.Message -match "powershell.exe")
} | Format-List TimeCreated, Message
```

# Detection Opportunities

Security teams should monitor for:

- WMIC spawning PowerShell
- WMIC spawning CMD
- WMIC spawning Script Hosts
- Hidden PowerShell execution
- WMI process creation outside administrative maintenance windows

# Lessons Learned

- WMI is a legitimate Windows administration framework frequently abused by attackers.
- Sysmon Event ID 1 provides valuable visibility into process creation.
- WMIC launching PowerShell is uncommon in normal user activity and should be investigated.
- Combining Sysmon telemetry with Wazuh significantly improves detection and investigation capabilities.
